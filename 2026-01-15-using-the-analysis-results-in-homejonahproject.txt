     PatternFly 5 to 6 Migration Plan

     Overview

     Migrate the konveyor-static-report project from PatternFly 5 to
     PatternFly 6. The Konveyor analysis tool has identified 13 violation 
     types across 100+ code locations in 24 files.

     Migration Strategy: Hybrid approach combining automated codemods with
     manual updates
     React Version: Keep React 17.0.2 (PF6 supports it; no need to upgrade)
     Estimated Time: 7-9 hours

     Critical Files

     - package.json - Update all PatternFly dependencies to v6
     - src/shared/hooks/useToolbar/useToolbar.ts - ToolbarChip →
     ToolbarLabel (affects all filtering)
     - src/layout/sidebar.tsx - MANDATORY: Remove theme prop from
     PageSidebar
     - src/layout/header.tsx - Multiple breaking changes (MastheadBrand,
     breakpoint units, prop renames)
     - src/shared/components/violations-table/violations-table.tsx - Largest
      component demonstrating all patterns
     - CSS files: index.css, App.css, simple-context.css,
     application-list.css

     Implementation Steps

     Phase 1: Update Dependencies

     1. Update package.json

     Replace PatternFly 5 packages with v6:
     "@patternfly/patternfly": "^6.0.0",

     "@patternfly/react-table": "^6.0.0",
     "@patternfly/react-charts": "^8.0.0",
     "@patternfly/react-code-editor": "^6.0.0"

     2. Install dependencies
     npm install

     Build will fail at this point - expected.

     Phase 2: Run Automated Codemods

     Run PatternFly's official codemods to handle mechanical changes:

     # Update CSS class names (pf-v5 → pf-v6)
     npx @patternfly/pf-codemods class-name-updater --v6 --fix src/

     # Update React tokens and CSS variables in TS/TSX
     npx @patternfly/pf-codemods tokens-update --fix src/

     # Update CSS variables in stylesheets
     npx @patternfly/pf-codemods css-vars-updater --fix src/

     Review git diff after codemods run. Look for "hot pink" markers
     indicating unresolved values.
